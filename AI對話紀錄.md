## 🤖 Claude

這是 **textarea（簡短介紹）的即時字數計數器**，做三件事：

**1. 顯示目前字數 / 上限**
即時顯示「現在打了幾個字 / 最多 100 字」，例如：`23 / 100`、`87 / 100`。

`saveData()` 裡 `rest` 指的是？

---

## 🤖 Claude

`rest` 是 JavaScript 的「解構賦值 + 其餘運算子 (rest operator)」用法，意思是「**剩下的全部屬性**」。

這是一個箭頭函式，做的事是：把物件拆開 → 拿掉 `expanded` 和 `imgError` → **剩下的全部丟進 `rest`** → 然後 return `rest`

假設有一筆資料包含 id、name、category、image、description、rating、createdAt、expanded、imgError，經過處理後：
- `expanded` 被抽出 → 丟掉
- `imgError` 被抽出 → 丟掉
- `rest` 接住剩下的全部欄位

**為什麼要這樣做？**

`expanded`（是否展開報告）和 `imgError`（圖片是否載入失敗）是 UI 狀態，不需要存進 localStorage。存進去也沒意義，還浪費空間，所以**先拆掉再存**。

**`...` 的兩種用法（容易搞混）**

| 寫法 | 名稱 | 作用 |
|------|------|------|
| `const { a, ...rest } = obj` | **Rest**（收集剩下的） | 把剩下屬性打包進 rest |
| `const newObj = { ...obj, b: 2 }` | **Spread**（展開） | 把 obj 屬性攤平到新物件 |

語法一樣，但位置不同意義不同：在 `=` 左邊或函式參數 → Rest（收）；在 `=` 右邊或字面值裡 → Spread（展）。

## 🤖 Claude

這是**推薦程度的 5 顆星評分按鈕**，逐行重點：

- **`v-for="n in 5"`**：Vue 迴圈，n 依序是 1~5，注意從 1 開始（不是 0）
- **`type="button"`**：因為在 form 裡，不加的話預設是 submit，點下去會送出表單
- **`role="radio"`**：語意上是單選，告訴螢幕閱讀器當作 radio 按鈕處理
- **`:aria-checked`**：標記哪顆星是選中的，給螢幕閱讀器讀
- **`:tabindex`**：只有「當前選中的那顆」tabindex=0（可 Tab），其他是 -1（Tab 跳過），符合 W3C radio group 標準操作模式
- **`:class="{ filled: n <= form.rating }"`**：n 小於等於分數時加 filled class，讓星星變亮（例如評 3 分 → 前 3 顆亮）
- **`:aria-label`**：按鈕內容是 ★ 符號，螢幕閱讀器念不出來，用 aria-label 補上「1 顆星」、「2 顆星」
- **`@click="form.rating = n"`**：點第 n 顆星 → 設定分數

為什麼這麼複雜？純視覺只要 `@click` 就夠，但為了 WCAG A + AA，必須同時做到：鍵盤可操作、螢幕閱讀器可讀、視覺狀態回饋、不破壞表單行為。

## 🤖 Claude

這是**「調查摘要」的多行輸入框**，屬性分類：

**Vue 動態綁定（有 `:` 或 `v-` 開頭）**
- **`v-model.trim`**：雙向綁定 + 自動去頭尾空白（避免空白佔字數又難驗證）
- **`:class="{ 'is-error': errors.description }"`**：驗證失敗時加紅框 class（`is-error` 要加引號因為含 `-`）
- **`:aria-invalid="!!errors.description"`**：`!!` 把任何值轉成 true/false，`aria-invalid` 只接受布林值，不能是字串
- **`:maxlength`**：綁定 LIMITS.DESC_MAX，改常數自動同步
- **`:placeholder`**：用樣板字串 `` ` `` 插入 LIMITS.DESC_MAX，改常數自動同步

**HTML 原生屬性（無前綴）**
- **`id="f-desc"`**：`f-` 前綴代表 form 欄位，對應 `<label for="f-desc">`
- **`aria-describedby="err-desc desc-count"`**：關聯兩個輔助說明元素（錯誤訊息 + 字數計數），螢幕閱讀器聚焦時一起念出，兩個 id 用**空格**分隔
- **`rows="3"`**：初始顯示 3 行高，不是字數限制

三種狀態：正常（灰色邊框）→ 輸入中（顯示字數）→ 送出驗證失敗（紅框 + 錯誤訊息）。

## 🤖 Claude

這是無障礙屬性，告訴螢幕閱讀器「這個區塊的標題是哪個元素」。

`aria-labelledby="form-title"` 意思是：這個 `<section>` 的標籤，請參考 id 為 `form-title` 的元素內容。螢幕閱讀器讀到這個 section 時，會念出對應 h2 的文字，讓視障使用者知道區塊名稱並可快速跳轉。

**`aria-labelledby` vs `aria-label`**

| 屬性 | 寫法 | 用在什麼情況 |
|---|---|---|
| `aria-labelledby` | 指向頁面上已存在的元素 id | 頁面上已有標題文字，重複利用 |
| `aria-label` | 直接寫字串 | 頁面上沒有可見標題 |

**特殊案例：`stats-title` 是視覺隱藏的**

統計區的 h2 套用 `visually-hidden` class（CSS position absolute + width/height 1px），視覺上不佔版面，但螢幕閱讀器讀得到。這樣的組合技：眼睛看的人不會看到多餘標題，用螢幕閱讀器的人聽得到區塊名稱。
